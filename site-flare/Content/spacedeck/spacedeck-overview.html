SpaceDeck

Overview

SpaceDeck is an intuitive, streamlined user interface to manage and control the GigaSpaces environment.

Using SpaceDeck, you can define the tools to bring legacy System of Record (SoR) databases into the in-memory data grid that is the core of the GigaSpaces system.

 

SpaceDeck also includes a set of command line interface (CLI) tools to manage the GigaSpaces system.

SpaceDeck has the following features:

  • No-code functionality – the environment can be set up and monitored using the user-friendly SpaceDeck screens, eliminating the need for writing or customizing code

  • The ability for you to create and execute low code data services.

  • A full complement of data and system monitoring tools, built into the environment

This version of SpaceDeck does not yet cover all functionality. On Platform Feature Access page explains which of our UIs manage each of our main platform operations. SpaceDeck is initially available as part of our Smart DIH Kubernetes installation.

Who Can Use SpaceDeck?

SpaceDeck can be used by various user types ("personas") as follows:

System Integrators – integrate data from Systems of Record into the Smart DIH, using data pipelines

Data Engineers – study the data unified in a Space using data queries. Integrate data from Systems of Record into the Smart DIH, using data pipelines

System Admins – manage day-to-day operations of the system; inspect and monitor its components

Why Do I Need SpaceDeck?

  • If you are running applications that require access to heterogeneous data that is spread across various Systems of Record – you need to gather that data into a unified data model that is not dependent on any particular database or operating system.

  • If you want to make that data instantly accessible – including querying and updating – in a proven in-memory data grid that is scalable, automatically backed up, designed for the Kubernetes environment, and has easy access to microservices, then you need Smart DIH.

What Actions Do I Take with SpaceDeck?

Using SpaceDeck, you can:

Define and Modify Data - which data will be pulled from the System of Record and placed in the GigaSpaces in-memory data grid or in SSD storage. This data will then be available for fast read/write access. Note that in-memory and SSD data is constantly synchronized with the SoR data.

This definition process includes defining the Data Sources (databases), specifying which tables and fields will be pulled in-memory, and includes additional options to minimize memory footprint and prioritize high-use data.

Monitor the GigaSpaces system, viewing transaction times and counts, spotting data bottlenecks, and ensuring the smooth 24/7 operation of the system.

Using SpaceDeck, you can define the following items:

  • Space -

  • Data Source – the database in the System of Record that will be used as a source and target of data for the Data Pipeline

  • Data Pipeline – the mechanism to transfer data to and from selected tables in the Data Source to selected Space objects in the GigaSpace in-memory data grid

SpaceDeck also provide monitoring and query screens to view the system status:

  • Spaces Status screen – show status and real-time monitoring of the Spaces in the data grid

  • Data Pipeline Monitoring screen – view Data Pipeline statuses, and stop/start pipelines

  • Data Query screen – perform SQL queries against the data in a Space

 

Advantages of Data Pipelines include:

  • Flexible extract of data from multiple databases and tables

  • Import of data into the high-speed GigaSpaces in-memory

  • Unified data model for read/write of pipeline data

  • A metrics dashboard showing data activity, including data volumes and statistics and any data validation errors or other access errors.
