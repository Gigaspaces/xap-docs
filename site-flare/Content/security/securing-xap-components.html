Securing Components

Each major component in (data grid, Grid Service Agent, Grid Service Manager, Grid Service Container, Processing Unit, and Web Management Console) has its own security configuration that can be adapted individually to suit the needs of your application environment. Security can also be enabled for other components and processes in the environment. This section explains how security relates to each component, and how to configure the properties to properly secure your application.

You can apply security settings at the following levels:

  • Data - You can secure the Space, Processing Unit, local cache/view, Space filters, task execution/executors, event-driven remoting, and the JDBC driver.
  • Service Grid - You can enable security for the Grid Service Agent (GSA), Grid Service Manager (GSM), and Grid Service Container (GSC).
  • Transport layer - contains a generic network filter that also provides SSL support via an SSL communication filter.

To administer and manage these components, set security configurations for:

  • Web Management Console - using a secured Web-UI server.
  • REST Manager API - using a secured RESTful API.
