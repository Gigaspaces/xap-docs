GigaSpaces SpaceDeck Overview

Why Do I Need SpaceDeck?

  • If you are running applications that require access to heterogeneous data that is spread across various Systems of Record – you need to gather that data into a unified data model that is not dependent on any particular database or operating system.

  • If you want to make that data instantly accessible – including querying and updating – in a proven in-memory data grid that is scalable, automatically backed up, designed for the Kubernetes environment, and has easy access to microservices, then you need GigaSpaces DIH.

  • If you want to design and manage that data using a no-code, elegant, beyond state-of-the-art user interface, then you need SpaceDeck.

What is SpaceDeck?

SpaceDeck is a user-friendly web interface that will provide full functionality and control of the GigaSpaces environment.

SpaceDeck has the following features:

  • No-code functionality – the environment can be set up and monitored using the user-friendly SpaceDeck screens, without the necessity of writing or customizing code.

  • Crystal-clear, step-by-step instructions for defining and running the online environment

  • A full complement of data and system monitoring tools, built in to the environment

This introductory version does not cover all functionality and is intended as an introduction to the new UI and a supplement to our existing Ops Manager, GigaSpaces Management Center and Web Management Console UIs. SpaceDeck is initially available for SmartDIH installations, and will be extended in future releases to eventually replace the older UIs.

Using SpaceDeck, the user can define the following items:

  • Data Source – the database in the System of Record that will be used as a source and target of data for the data pipeline

  • Data Pipeline – the mechanism to transfer data to and from selected tables in the data source to selected Space objects in the GigaSpace in-memory data grid

SpaceDeck also provide a monitoring screens to view the system status:

  • Spaces Status screen – show status and real-time monitoring of the Spaces in the data grid

  • Data pipeline monitoring screen – view data pipeline statuses, and stop/start pipelines

  • Data Query screen – perform SQL queries against the data in a Space

What are the Steps to Using SpaceDeck?

After you have installed the basic GigaSpaces DIH environment, you can use SpaceDeck to perform the following steps.

Define the data sources from the System of Record

1. Define the Data Sources from the System or Record 2. Define a Data Pipeline 3. Monitor the activity of the Data Pipeline 4. Perform SQL queries on Space/Pipeline data
Specify URLs for the SoR databases in SpaceDeck – SpaceDeck – Data Sources Specify which Data Source feeds the Data Pipeline, and which in-memory Space will contain the data – SpaceDeck – Data Pipeline – Create New Pipeline View dynamic metrics and start/stop the pipeline – SpaceDeck – Data Pipeline – Monitoring Pipelines Use supported SQL-99 commands – SpaceDeck – Data Query
