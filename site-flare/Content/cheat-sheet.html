Cheat Sheet for GigaSpaces Documentation

Styles (Lists, Code, Images)

We apply styles to paragraphs in order to format text. Use the following styles when you write or update documentation.

Headings

We generally use three heading styles (h1, h2, and h3). Headings are used to indicate a hierarchical order to information. Don't use a heading style just to emphasize text.

HTML:

<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>

Images

There are multiple image styles so that you can assign a size to an image. This will ensure that it sits nicely on the page, or in the line of text where you want it to appear.

Unsized image:

HTML:

Image is set to the maximum width of the natural image size or the max-width of the viewport (area of the website page where it appears):

HTML:

 <p><img src="Resources/Static/attachment_files/danger-noodle.jpg" class="tc-picture100" /></p>

Image is set to 80% of the natural image size or the width of the viewport (area of the website page where it appears):

HTML:

Image is set to 50% of the natural image size or the width of the viewport (area of the website page where it appears):

Image is set to 30% of the natural image size or the width of the viewport (area of the website page where it appears):

Spans (Local Formatting)

Use a span to apply local formatting to words within a paragraph, like bold and italic.

Bold: Bold, Italic: Italic, Monospace: monospace

HTML:

 <p>Bold: <span class="tc-bold">Bold</span>, Italic: <span class="tc-italic">Italic</span>, Monospace: <code>monospace</code></p>

Colored text: blue text, green text, red text

HTML:

<p>Colored text: <span class="tc-textblue">blue text</span>, <span class="tc-textgreen">green text</span>, <span class="tc-textred">red text</span></p>

Text with awesomefont icons - can be used in tables instead of the Note and Important boxes (apply the span to the entire line of text and the icon will display at the beginning):

Versioning (Conditions and Variables)

We use conditions instead of branching the documentation for each new version. We use variables instead of short codes to keep things like version numbers and product/feature names up to date.

Conditions

If you are adding information that is version specific, you must make sure to condition it for that version. You can apply a condition to a word, paragraph, or a complete topic. We use the following methodology to condition per version - born (introduced in this version), died (removed in this version), and only (specific to this version).

This is what you added because of new development.

HTML:

<p>This is <MadCap:conditionalText MadCap:conditions="Version.14-2-born">what you added </MadCap:conditionalText> because of new development.</p>

This is what you are hiding in this version because it was removed.

HTML:

<p>This is <MadCap:conditionalText MadCap:conditions="Version.14-2-died">what you are hiding in this version</MadCap:conditionalText> because it was removed.</p>

This is how to condition a paragraph in a topic.

HTML:

<p MadCap:conditions="Version.14-0-only">This is how to condition a paragraph in a topic. </p>

Note the differences between conditioning a portion of tagged content vs. conditioning content that is fully enclosed by a tag (such as a paragraph, table row, or div). When conditioning content that is enclosed in its own tag, add this to the opening tag: MadCap:conditions="<condition definition>". Do not add a closing tag.

Variables

Don't hard-code things like software versions in the text or in code samples (for third-party software or other items that are included in our product package).

Add the variable in the sentence, for example release .

HTML:

<p>Add the variable in the sentence, for example <MadCap:variable name="General.ProductNameIE" /> release <MadCap:variable name="Versions.product-version-short" />.</p>

For a complete list of version variables and their values, see the Versions.flvar file.

Links

Use hyperlinks for external links (to the Github, other websites, etc.), and cross references for internal links (to another topic on the doc website).

Hyperlink that opens in a new browser tab: This is a cool external link.

HTML:

<p>This is a cool <a href="http://insult.dream40.org/" target="_blank">external link</a>.</p>

Cross reference that opens a different page of the doc website in the same tab: Cheat Sheet 2

HTML:

<p>Cross reference to different page of the doc website: <MadCap:xref href="cheat-sheet2.html"> Cheat Sheet 2</MadCap:xref></p>

To add a link to a JIRA, update the JIRA-Id in the MadCap:snippetVarables:

HTML:
