What is the Smart Portfolio?

Smart Portfolio — SmartCache, SmartODS and Smart Augmented Transactions — is a partitioned, distributed in-memory data store and analytics system that dramatically improves the performance of caching, Operational Data Stores, and transactions.

Product Architecture

uses an in-memory core architecture developed with a Space-based design model. design components include the following:

  • Space-Based Remoting
    Remote service invocations using Jini Network Architecture that provide location transparency for enterprise-wide data access

  • Core API and In-Memory Data Grid
    Direct access to a RAM-based data grid or Space

  • Messaging and Events
    Object-based event processing, based on and extending the properties of Java EE message-driven beans

  • Integrations
    Integration capabilities with non- components

  • Data Partitioning
    Load balancing via data partitioning allows code and data to be spread across many individual Processing Units, located in a heterogeneous mix of server instances on on-prem and cloud machines.
    Smart data partitioning locates frequently-access data in the same partition as the code that uses the data (co-location), and allows horizontal and vertical scaling to perform dynamic, rule-based scaling of the system.

See Data Grid Architecture for more details of the product architecture.

Components of the Smart Portfolio

consists of three products: SmartCache, SmartODS and Smart Augmented Transactions.

  • SmartCache provides fast, scalable cache from any data store.

  • SmartODS is an integrated, enterprise-wide data store that allows you to create a new Operational Data Store (Digital Integration Hub) or augment your existing data store.

  • Smart Augmented Transactions provides translytical processing for fast access, updating and analysis of enterprise-critical information.

Smart Portfolio meets the following critical design criteria:

  • Advanced memory management

    • High-speed in-memory data grid with millisecond-range response time

    • Multi-tiered data storage across RAM, SSD, and Storage-Class Memory (3DXPoint)

    • Compressed RAM footprint

    • Optimized data co-location for fast Join performance

  • Loss-less data technologies

    • Highly available, always-on data backup

    • Disaster recovery via geo-redundancy, fast data replication and native persistence

    • Data persistency with low latency and strong consistency

    • Atomic, transaction-based processing

  • Support for complex SQL-99 compliant queries

  • Enterprise-scale, flexible data management:

    • Distributed processing partitions, with Wide Area Network support

    • Automatic, trigger-driven vertical and horizontal scaling

  • Minimally-invasive installation

can be installed on-prem, in the cloud, or in a hybrid environment.
