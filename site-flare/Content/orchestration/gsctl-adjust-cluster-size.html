Adjusting the Size of a Cluster

 

While ElasticGrid performs auto-scaling so that the cluster adjusts its size automatically to adapt to the current workload, you can also change the size manually if necessary by adding or removing nodes.

Adding a Node

Follow the steps in the procedure to add a node to a cluster.

To add a node:

  1. Type the following command:

    java -jar gsctl.jar add-node
  2. Download the most current version of the gsctl.jar file from Github to the directory you created (gsDemo) and extract the contents.

  3. To create a cluster, run the following command:

    java -jar gsctl.jar create

    The cloud platform begins to create the cluster:

    You can monitor the progress of the cluster in your cloud platform dashboard, for example the VPC Dashboard in AWS:

    The process of creating the cluster takes only a few minutes until the nodes are up and running. You can see when the Master (server) nodes and Worker (client) nodes are running and have passed the status checks.
