GigaOps Stack

When using the simplest orchestration options, implementing products involves several activities:

  • Creating and maintaining the infrastructure (on-premise or cloud servers), starting with:
    1. Downloading and installing on every server
    2. Provisioning and editing the configuration files (for example, applying the license, defining environment variables, and defining the lookup service)
    3. Running the platform and monitoring it for problems or failure
  • Developing applications on top of
  • Periodic upgrade of and/or the hosted applications

Organizations that have the resources may develop a script or a tool that handles the installation, provisioning and monitoring, otherwise the orchestration is completely manual.

A simpler and more robust level of orchestration is to use Kubernetes, which is an open-source container system for automating application deployment, scaling, and management. If you have a Kubernetes cluster, deploying and your applications is relatively easy. However, although cloud providers allow running a Kubernetes cluster, provisioning and maintaining an on-premise cluster can be difficult and complex, especially regarding configuration. In addition, while developing on Kubernetes is relatively simple, the situation can get complex when you're ready to expose your application to the outside world.

The GigaOps Stack reduces the complexity of all aspects required to implement products, proving a self-service cluster with end-to-end provisioning, ability to deploy new services, auto-recovery, self-monitoring, and tracing.

The GigaOps Stack is comprised of the following:

  • gsctl - a standalone CLI tool that you can use to create, provision, and install clusters in the cloud or on premise.
  • Ops Manager - the native administration and monitoring tool, which is web based and runs automatically when services are deployed.
  • Monitoring stack - based on InfluxDB, Grafana, and Telegraf for monitoring cluster activity and metrics using dashboards
  • ElastiGrid or Kubernetes orchestration
