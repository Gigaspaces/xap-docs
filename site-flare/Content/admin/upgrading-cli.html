Upgrading the CLI

The new Command Line Interface which was introduced in release 12.3 provides a simpler syntax and is cloud-friendly (based on the REST Manager API).

The legacy CLI is still supported, but has been deprecated as of and release 14.0 and will be removed in a future version. Customers upgrading from older product versions are encouraged to upgrade to the new CLI.

To get started with the new CLI, simply run xap --help gs --help from the product's bin folder.

Setting the CLI Version

We use versioning to formally differentiate between the CLIs: the ‘new’ CLI version is 2, and the ‘legacy’ CLI version is 1.

Until release 14.5, running the insightedge or xap script started CLI v2 whereas running gs started CLI v1. Starting in release 14.5, running gs starts the latest version of the CLI (v2), and the insightedge & xap scripts have been removed.

To explicitly set the CLI version use one of the following options:

  • Global - Edit the setenv-overrides script (located at the product's bin folder) and set the XAP_CLI_VERSION environment variable to 1.

  • Per-command - Use the --cli-version option to set the CLI version for a specific command invocation, such as gs --cli-version=1 . This option overrides the XAP_CLI_VERSION environment variable, if set.

If you’ve just started upgrading from an old version, you’ll probably want to use XAP_CLI_VERSION,since it globally reverts to the old syntax and your scripts can run unchanged. Since the --cli-version option overrides the environment variable, you can migrate your scripts gradually by changing one command at a time to use and test the new syntax, before moving to the next command.

Note that version 1 of the CLI is deprecated and will be removed in the future - the versioning mechanism is provided as an aide to simplify migration, but should not be used as a long-term solution.

If your scripts were already using CLI v2 using the insightedge & xap scripts, simply revise them to call gs instead. If you’re interested, the deprecated-scripts.zip archive at the product's bin folder contains revised insightedge & xap scripts which simply invoke gs --cli-version=2 with the specified arguments on your behalf.

Obsolete Commands

This section lists commands that have not been ported to the new Command Line Interface for various reasons.

Interactive Shell Commands

The old Command Line Interface supports an interactive shell mode, and as such includes a set of shell-related commands, which are currently not supported in the new CLI as it is not interactive.

  • cd
  • dir
  • pwd
  • ls
  • set
  • login

Elastic Processing Unit Commands

The Elastic Processing Unit was deprecated in version 12.1, therefore its commands were not ported to the new Command Line Interface.

  • deploy-elastic-space
  • deploy-elastic-pu
  • scale

Miscellaneous

  • pudeploy - alias for the deploy command.
  • deploy-memcached - Can be done via the deploy command.
  • deploy-rest - Can be done via the deploy command.
  • deploy-application and undeploy-application - Low usage rate, can be done via the deploy/undeploy commands and scripting.
  • jconsole - The new CLI is based on the REST protocol, so this command is misleading. Users may access the standard JConsole directly.
  • admin multicastTest - The new CLI is based on the Manager, which uses unicast.
