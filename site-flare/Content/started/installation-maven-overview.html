Maven Artifacts

Maven is a tool used to automate and simplify the development cycle of any Java-based project. The plugin for Maven utilizes Maven to simplify the development cycle of -based applications. This section describes the Maven artifacts that are provided, and how to get started with the data grid using Maven.. It also describes how to install and use the XAP Maven plugin.

Installing Maven Artifacts

Prerequisites

Maven must be installed and included in your system PATH variable.

If you don't have Maven installed, refer to the Apache Maven documentation for download instructions.

Installation

To install the Maven artifacts, run the following command:

 maven install
 maven install

Advanced Installation Options

Some Maven setups require a slightly different Maven installation process. 's Maven installation is designed to be customized if needed, and works as follows:

  1. Create a temporary directory (/config/maven/installer), and a main pom.xml for the installation process.

  2. Add each artifact in /config/maven/gs-artifacts.txt to the main pom.xml, and extract the artifact's pom.xml if needed.

  3. Install the main pom.xml using a standard mvn install command.

  4. Delete the temporary folder.

If needed, you can customize the list of installed artifacts by editing the gs-artifacts.txt file. In addition, the gs maven install command supports an optional --generate-only flag, which generates the temporary directory and files without installation (steps 1+2), so you can tailor the generated content to your needs.

Maven Repository

artifacts are not currently published in the Maven Central Repo, so you must first configure a repository:

<repository>
   <id>org.openspaces</id>
   <url>http://maven-repository.openspaces.org</url>
</repository>

Artifacts

The main dependency required to use the data grid is xap-openspaces.

<dependency>
  <groupId>org.gigaspaces</groupId>
  <artifactId>xap-openspaces</artifactId>
  <version>[%=Versions.maven-version-MX%]</version>
</dependency>
