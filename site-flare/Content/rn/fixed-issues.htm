Changelog

This section lists the changes that were made in InsightEdge Platform 14.0.x.

14.0.0

Features and Enhancements

  • : Demote Primary Space to Backup.
  • : Kubernetes support.
  • : New MemoryXtend driver for Persistent Memory (PMEM).
  • : Change InsightEdge space name in demo mode.
  • : Migrate InsightEdge examples from separate repo to InsightEdge.
  • : Upgrade Jetty version to 9.3 latest.
  • : Deprecate gs-agent script.
  • : Deprecate legacy command line interface.
  • : Support querying space data from REST Manager and CLI.
  • : Enhanced CLI "pu run' command validation to include invalid instances.
  • : Improve error message for CLI command "pu run' when file not found.
  • : Support unmanaged Processing Units in Admin API.
  • : Changed default rownum from 5000 to 10 in Web Management Console.
  • : Upgrade Zeppelin to 0.8.
  • : Add Zeppelin InsightEdge JDBC interpreter.
  • : Add autoCommit property to InsightEdge JDBC Driver (for Zeppelin).
  • : New API to initialize InsightEdge from existing Spark context/session.
  • : Increase ZooKeeper client session timeout setting to 15 seconds.
  • : Enhanced XAP.NET to support dtd validation via Tls12 url (s3).
  • : End Of Life - XAP.NET for .NET v3.5.
  • : InsightEdge JDBC driver automatically creates an alias for types using short name if not ambigous.
  • : InsightEdge JDBC driver automatically creates an alias for types using underscores to allow non-escaped usage.
  • : End Of Life - GigaSpace.getModifiersForIsolationLevel().
  • : Deprecation - GSIterator and GigaSpace.iterator().
  • : Enhance Deploy command in CLI to auto-zip resource in case it is a directory.
  • : Reduce Web Management Console (web-ui) memory consumption due to unused statistics.
  • : Upgrade Spark to 2.3.2.
  • : Upgrade to Spring Security 4.2.8 and Spring Framework 4.3.19.
  • : Configure insightedge-submit script to use Platform Manager name instead of lookup locator when running on Kubernetes.
  • : Reduce replication network traffic by skipping discarded packets.
  • : Add ChangeSet.isEmpty() to indicate if it contains any change operations.
  • : Override the IdQuery.equals() default implementation to streamline unit testing.
  • : Expose Jetty ports in the Admin API (such as the SslSelectChannelConnector used for HTTPS).
  • : Run InsightEdge Spark jobs on Kubernetes using the InsightEdge Docker image.
  • : Add system property to show standalone Spaces in REST Manager API and CLI.
  • : Enhance logging to reflect replication connection state changes.
  • : Enhance command line --server option to support host:port format.
  • : Add a summary to CLI output.
  • : Improve XAP and InsightEdge demo command behavior when XAP_MANAGER_SERVERS is configured.

Resolved Issues

  • : InsightEdge JDBC driver does not work with Spark SQL.
  • : When a transaction aborts due to network failure, it may not release the participating entries immediately.
  • : Misspelled License in "Licence data is corrupted" error message.
  • : Typo in XAP Manager system property for configuring ZooKeeper session timeout.
  • : Fix InsightEdge JDBC Driver initialization to return null when connection fails.
  • : EXT_JAVA_OPTIONS environment variable is ignored when using OpenJDK with Microsoft Windows.
  • : SPARK_MOUNTED_CLASSPATH contains incorrect semicolon on Microsoft Windows.
  • : Null comparison does not work in SQL query when using a nested collection of objects.
  • : SQL query with Java date/time WHERE clause fails when date/time format property is configured.
  • : The Web Management Console fails to parse java.util.Date properties.
  • : Can't generate dump files from user interface after first attempt.
  • : "Only Live Services Log Dump" doesn't work as expected.

The complete list of changes is also available in JIRA.
