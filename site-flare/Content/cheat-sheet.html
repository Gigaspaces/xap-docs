Cheat Sheet for GigaSpaces Documentation

Styles

We apply styles to paragraphs in order to format text. Use the following styles when you write or update documentation.

Headings

We use three heading styles (h1, h2, and h3 -- see the headings above). Headings are used to indicate a hierarchical order to information. Don't use a heading style just to emphasize text.

Conditions and Variables

We use conditions instead of branching the documentation for each new version. We use variables instead of short codes to keep things like version numbers and product/feature names up to date.

Conditions

If you are adding information that is version specific, you must make sure to condition it for that version. You can apply a condition to a word, paragraph, or a complete topic. We use the following methodology to condition per version - born (introduced in this version), died (removed in this version), and only (specific to this version).

This is how to condition content that you added to a sentence or content that you need to remove from a sentence.

This is how to condition a paragraph in a topic.

Variables

Don't hard-code things like software versions in the text or in code samples (for things that are included in our product package).

Add the variable in the body of the sentence, for example InsightEdge release .

For a complete list of version variables and their values, see the Versions.flvar file.
