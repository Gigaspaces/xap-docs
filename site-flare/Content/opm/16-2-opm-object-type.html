Object Type

Within this page, the IT administrator or developer can review the Object Type properties and metrics related to size, distribution per partition etc.

At the top of the page, it is possible to review the Object Type name and data related to the number of entries, indexes, and Notify Contains generated by the Object Type.

Data distribution per partition

  • Number of entries - review the Object Type data distribution between the deployed partitions of the service. It is possible to review the number of entries per partition and identify situations where the data partitioning is unbalanced.
  • Number of reads - review the distribution of application reads to the Object Type and split between the deployed partitions.

These two charts provide real-time indications and better understanding for users of their data distribution and use.

Object Type properties

Enable the user to review the implemented properties as part of an Object Type including their types. In addition, it is possible to review indications related to routing key and indexes

Object Type Indexes

By clicking on the indexes tab, it is possible to review the list of indexes defined for the Object Type. It is possible to review the used properties per each index and its type - e.g. regular, composite etc.

Object Type SQL Editor

If we open the SQL Editor tab, a SQL SELECT statement is displayed that selects the selected Object Type, in this case the Person Object.

Press Run Query. The results will appear in the Query Results table on the bottom of the page.

The user can edit the SQL query, for example:

SELECT uid,* FROM com.mycompany.app.model.Person WHERE rowNum <= 10 AND lastname = 'Smith'

For a Document object with nested properties, you can code:

SELECT Address:city_name as city_name FROM com.mycompany.app.model.Person
where the field city_name is a property of the Address object.
