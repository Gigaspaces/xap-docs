ElastiGrid

This section describes how to deploy in cloud-native environments using the GigaOps stack, which is comprised of the following:

  • ElastiGrid - orchestration option for cloud, on-premise, or hybrid environments that provides automated deployment, scaling, and management of virtualized and containerized applications.
  • gsctl - a standalone CLI tool that you can use to create, provision, and install clusters in the cloud or on premise.
  • Ops Manager - the native administration and monitoring tool, which is web based and runs automatically when services are deployed.
  • Monitoring stack - based on InfluxDB, Grafana, and Telegraf for monitoring cluster activity and metrics using dashboards

how users use GigaSpaces?

Using our product is consisted of few steps:

  • Creating and maintaining the infrastructure (servers - on-premise/cloud)
  • Developing applications
  • Periodical upgrade of the product and/or the user’s applications

to install the product:

  1. Users need to download and install the product in every server they have
  2. Do some configurations by editing few files (e.g. license, lookups)
  3. Run our main component and monitor it in case it fails

Most likely the customer develops a script or a tool that does all of the above. This is the Ops part.

 

Two years ago, we added a support for Kubernetes which is an open-source container-orchestration system for automating application deployment, scaling, and management.

Given you have a Kubernetes cluster running, deploying our product and the user’s applications is relatively easy.

While the cloud providers allows running a Kubernetes cluster, having an on premise cluster can be hard and complex especially the configuration part.

Developing on Kubernetes can be complex when you need to expose your application to the outside world
