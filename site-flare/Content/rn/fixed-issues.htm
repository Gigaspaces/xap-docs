Changelog

This section lists the changes that were made in InsightEdge Platform 12.3.x.

12.3.1

This service pack was released in July 2018.

Improvements

  • : Sort results for REST operations related to Spaces and Processing Units.
  • : CLI Auto Complete.
  • : Simplified the CLI command "pu decrement" to receive only one parameter: pui_id.
  • : Enhance CLI output to display "N/A" instead of "-1" where relevant.
  • : Upgrade Apache Curator to 4.0.1.
  • : Improve memory allocation by pre-allocating ArrayList instances in the SQL execution engine.
  • : Upgrade RocksDB to 5.11.3.
  • : Display blobstore operation statistics in the Web Management Console.
  • : Enable configuring the MINIMAL_BUFFER_DIFF_TO_ALLOCATE property of the OffHeapMemoryPool.
  • : Share a single instance of Curator Framework client to support large-scale deployments.
  • : Handle Apache Curator leader election events in a single threaded queue.
  • : Configure Spark to include InsightEdge dependencies on startup.
  • : Upgrade Spring to 4.3.17.
  • : Upgrade Apache Zookeeper to 3.4.12 and Netty to 3.10.6.Final.
  • : Log fills up with "unmarshalling failure" messages when a cluster partition member is unreachable.
  • : XAP and InsightEdge demo command should fail if XAP_MANAGER_SERVERS is configured.
  • : Enable building Python notebook in build process for InsightEdge Zeppelin tutorial.
  • : Upgrade Apache Spark to version 2.3.1.
  • : Enhance REST API and CLI to support primary zones.
  • : Display redo log information in the Web Management Console.
  • : Remove dashes from new CLI output to simplify output parsing.
  • : Support public and private IP addresses for Docker deployment.
  • : Decouple JPA integration from OpenSpaces module.

Resolved Issues

  • : Could not display the off-heap columns in the Web Management Console (in systems with MemoryXtend).
  • : Running the demo command in the CLI using off-heap generates a NoClassDefFoundError.
  • : Provisioning a Space instance fails on "Identified another participant with the same name for Space".
  • : Index intersection optimization did not work as expected.
  • : Insightedge CLI command fails to start Spark master and Spark worker when XAP_MANAGER_SERVERS is not configured.
  • : Unhealthy Space remains in Stopped state indefinitely after a network failure.
  • : Apache ZooKeeper sometimes fails to create a data directory on startup.
  • : Space instance recovery fails on "Failed while getting participants from zookeeper server".
  • : IllegalArgumentException: A metric named process_cpu_time-total already exists.
  • : Space instance recovery fails on "Failed while getting participants from zookeeper server".
  • : IllegalArgumentException: A metric named process_cpu_time-total already exists.
  • : The SQL query "SELECT COUNT (...) GROUP BY" doesn't work on nested properties in the Web Management Console.
  • : The SQL query "SELECT COUNT (...) GROUP BY" doesn't work in the Web Management Console when it includes enums.
  • : Remoting service routes incorrectly when the routing key is long, and if the value is higher than the max integer.
  • : The XAP Manager mishandles pending requests when its leadership is relinquished.
  • : Storing the blobStoreVersion not implemented in MemoryXtend off-heap driver.
  • : The Java 8 LocalDateTime is not shown in the Web Management Console object inspector.
  • : The Replication Channels metrics for partition clusters are not being reported.
  • : ServerTypeDesc tree was not cloned correctly.
  • : Display blobstore operation statistics in the GigaSpaces Management Center.
  • : SpaceTypeDescriptorBuilder fails when explicitly indexing the routing key using deprecated values.
  • : Manager class path does not include cert file located in "com.gigaspaces.lib.opt.security' directory.
  • : Space instance becomes backup although it is offered leadership by Apache Zookeeper.
  • : Change operation replicated to mirror can cause NPE in certain failover scenarios.
  • : The xap space run CLI command fails when XAP_MANAGER_SERVERS is defined.
  • : Queries using the DISTINCT keyword do not work in the Web Management Console.
  • : Web Management Console displayed different values in the Monitoring tab and the Space statistics view for the number of data objects written to a Space.
  • : The "pu undeploy' CLI command returns an error when the resource JAR file is not found in the resources directory.
  • : The "xap pu run' CLI command might leave orphan sub-processes in some scenarios.

12.3.0

The main version was released in March 2018.

Features and Enhancements

  • : Unified Command Line Interface.
  • : New MemoryXtend storage driver for off-heap memory.
  • : When killing an external process, avoid SIGKILL on systems which do not support it (Windows).
  • : Docker Images for InsightEdge and XAP.
  • : Revise Space index types to provide option for reduced memory footprint.
  • : Reduce @SpaceId(autogenerate=false) memory footprint.
  • : Upgrade Apache Zookeeper to 3.4.10.
  • : Replication redolog compaction for mirror target.
  • : Created v2 of XAP Manager REST API.
  • : Renamed /deployments to /pus in v2 of XAP Manager REST API.
  • : Limit Lookup Locator Discovery interval to 1 minute instead of 1 hour.
  • : Change default of org.jini.rio.monitor.pendingRequestDelay to 1 minute instead of 10 minutes.
  • : Upgrade packaged Jetty server to 9.2.24.
  • : Upgrade Spring Framework to 4.3.13.
  • : Upgrade Spring Security to 4.2.3.
  • : Enhance gs-agent to Support running Web Management Console (â€“webui).
  • : Enhance CachePolicy configuration to use fluent syntax.
  • : Remove obsolete security algorithm to align with current standards.
  • : Added ability to monitor connection status for local views.
  • : Improved AbstractEventListenerContainer monitoring via public getter methods.
  • : Improve log message to include table name when initial load fails due to wrong column name.
  • : Improve responsiveness of remote statistics gathering in Admin API.
  • : Make instance variables in DefaultHibernateSpaceDataSourceConfigurer protected.
  • : User-defined cache criteria for hot data in MemoryXtend.
  • : Added more GSM state information to application dump.
  • : Reduce the footprint of the Lookup Service template cache.
  • : When the query result set includes only indexes, fetch results from off-heap memory instead of disk.
  • : Asynchronous execution should always return immediately.
  • : Add metrics to Lookup Service to facilitate troubleshooting of performance issues.
  • : Lookup Service (LUS) enhancements to improve stability in environments with large clusters.
  • : Enhance LRMI network filter buffer allocation strategy by adding support for large objects (>10MB).
  • : Skip login page if Web UI is in non-secured mode.

Resolved Issues

  • : Loop of restarts of PU instance in same container cause memory leak.
  • : Missing partition instance not provisioned after network disconnected and reconnected.
  • : Relocation listener is not called during failure to destroy instance.
  • : Elastic Processing Unit failed to fully deploy with service-limit=1 even when there were empty GSCs running.
  • : Eager Scale Strategy did not evenly redistribute partitions after a cluster node disconnected and reconnected.
  • : Use XapNetworkInfo.getHost() instead of InetAddress.getLocalHost().
  • : XAP Manager doesn't support com.gigaspaces.lib.opt.security system property for providing an alternative path for custom security JARs.
  • : Add missing xap-admin JAR for default XAP.NET application.
  • : Inefficient use of underlying data structure slows down iterator performance.
  • : In rare scenarios, a transactional Polling container may drop events.
  • : The Admin API blocks itself, becoming unresponsive and exploding memory usage.
  • : Incomplete recovery of processing units after healing of GSM (if there are 3 or more).
  • : Querying a type with enum on a re-deployed space returns incorrect results.
  • : Insightedge shutdown command on Mac systems doesn't terminate the gs-agent.
  • : Lookup service improvements resulted in loss of client notifications in rare scenarios.
  • : REST calls to Spark failed on machine failover.
  • : No proactive verification of user credentials when logging into secured web-ui.
  • : The REST manager specification contained the wrong request status values.
  • : Inconsistent Request ID type in Manager REST API (string vs. numeric).
  • : The ProcessingUnitInstance.relocationAndWait() may not abort after timeout, causing a permanent blocking state.

The complete list of changes is also available in JIRA.
