Starting on a Local Service Grid

This section explains how to start an environment on a local machine.

Prerequisites

Before running , verify that you have the following installed in your environment:

  • Java 8 or 11
  • Scala 2.11
  • Maven 3.1+

Starting the Local Environment

The environment consists of Spark and the data grid. You can use the Command Line Interface tool to start as described below.

Starting a Complete Environment

To start a complete environment on your local machine, run the following command from the /bin directory:

./  host run-agent --auto --gsc=2
 
  host run-agent --auto --gsc=2

This will start the following components on your machine:

  • Spark Master UI at localhost:8080
    • URL for submitting spark jobs: spark://127.0.0.1:7077
  • Spark Worker UI at localhost:8081
  • Zeppelin UI at localhost:9090
  • service grid:
    • Manager at localhost:8090
    • 2 Grid Service Containers (GSCs)
  • Web Management Console at localhost:8099

Then run the following command to deploy a Space called demo with 2 partitions on those GSCs:

./ space deploy --partitions=2 demo
 
  space deploy --partitions=2 demo

Stopping the Local Environment

To stop the local environment, use the kill-agent command that sends a kill signal to all relevant processes:

./ host kill-agent

   host kill-agent
