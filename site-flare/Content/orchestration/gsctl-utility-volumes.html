The GSCTL Utility for Persistent Data Using Amazon Volumes

Persisting services is useful in production environments because it allows us to reschedule a node without worrying about data loss.

In this topic you will learn how to deploy a cluster with volumes attached to services, supporting their life cycle.

Configuring services.yaml file

Now we must choose the service. We want to attach it a volume OR choose the sticky option to persist the data to the machine it runs on.

In case of sticky, the system should make a best-effort attempt to keep the allocation on the same machine and to ensure that the data will be kept through the whole life cycle.

Attaching the volume to the service

  • Enable persistency under constraints section

  • Provide volume

Services yaml configuration of persistency with volume example


				builtInServices:
				  grafana:
				    enabled: true
				    constraints:
				      targetComponents:
				      - "*"
				      targetProfiles:
				      - "*"
				      persistency:
				        enabled: true
				        sticky: false
				        volumes:
				        - "my-grafana"
				    properties: {}

Choosing sticky mode instead

  • Enable persistency under constraints section

  • Enable sticky

Services yaml configuration of persistency with sticky example



				builtInServices:
				  grafana:
				    enabled: true
				    constraints:
				      targetComponents:
				      - "*"
				      targetProfiles:
				      - "*"
				      persistency:
				        enabled: true
				        sticky: true
				        volumes: []
				    properties: {}

We can also target a profile or component that the service will run on: Choose “*” to allow placement on any profile/component (default).

Considerations

  • Note that currently we support persisting Grafana dashboards and the data they get from Telegraf which feeds InfluxDB — therefore only persisting InfluxDB and Grafana.

  • constraints available for all the services except mongoDB

  • note that manager service runs only on master nodes. All three of them.
