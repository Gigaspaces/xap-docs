Tableau Integration

This topic describes how to integrate Tableau with in a Microsoft Windows environment using the JDBC connector to retrieve data for visualization and business intelligence needs.

Prerequisites

Your host machine must have the following installed:

  • Java 8 or higher
  • Tableau version 2019.1 or higher

Integrating Tableau with

After setting up and starting , you have to add the JDBC connector to the Tableau Drivers folder. Scripts are provided that create the driver and invoke Tableau.

You can override the default configuration for these scripts by editing the variables in the gs-tableau-envin the product tools\tableau folder.

Creating a standalone JDBC Connector to Tableau

A standalone connector package is required for machines that access but don't have installed locally. To create the package ZIP file, run the gs-tableau-packagescript from the product tools\tableau folder. That file can be distributed and installed on any machine to access data using Tableau.

Accessing the Data Grid with Tableau

In order to use the Tableau connector, start Tableau using the gs-tableauscript from the product tools\tableau folder. The initial setup depends on your local environment.

  • If you don't have installed locally, install a standalone connector package ZIP file and then proceed to run the gs-tableauscript.
  • If you have installed locally, the JDBC connector is added by the gs-tableauscript. The first time you run the script the JDBC connector is automatically packaged and copied (this requires elevated permissions).

Exploring the JDBC Connector to Tableau

If you need to customize or troubleshoot the integration, these are the steps executed in the scripts.

  1. Run <XAP_HOME>\bin\insightedge maven install to install the Maven artifacts.
  2. Run <XAP_HOME>\insightedge\tools\jdbc\build-jdbc-client.cmd to create the insightedge-jdbc-client.jar file.
  3. Copy the file to the Tableau Drivers folder.

  4. gs-tableau runs the following command to start Tableau with the connector: 

    <TABLEAU_HOME>\bin\tableau.exe -DConnectPluginsPath=<GS_HOME>\insightedge\tools\jdbc

Configuring the Connection to

To configure the data source after Tableau is started:

  1. Click Connect in the Tableau desktop, and select Gigaspaces InsightEdge as the data source.

    If Gigaspaces InsightEdge does not appear in the Connect menu on first access ,click More and select Gigaspaces InsightEdge from the alphabetic list.

  2. In the Gigaspaces InsightEdge window, configure the options as follows:
    1. Type the Server host name or IP address. Include the port number if the Space is not using the default port.
    2. (Optional) If your system includes AnalyticsXtreme, you can type True for Enable AnalyticsXtreme.
    3. Type the Space name.
    4. (Optional) Add Advanced Parameters for the JDBC connection as key-value pairs.
    5. If your system is configured with a secure Space (See Security), type the Username and Password.

  3. Click Sign In to connect to the data grid.

  4. Verify that Tableau can see the Tables in your Space.

    A POJO in the data source must use java.sql.Date rather than java.util.Date.
