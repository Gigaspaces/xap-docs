SpaceDeck Installation and Operation

Installation

When deployed onto a Kubernetes cluster, SpaceDeck is automatically installed as part of the procedure in Deploying and Managing GigaSpaces in Kubernetes.

These instructions are for the Kubernetes environment. If you are using the Service Grid component of GigaSpaces, please contact your GigaSpaces representative for instructions how to install and run SpaceDeck.

Sample for docker-compose File


version: ‘2’
services
  space-deck:
    image: latest (e.g. gigaspaces/spacedeck:1.0.2)
    ports:
     - 4200:3000
    container_name: space-deck
    environment:
     - API_URL=http://di:8090/v2
     - DI_API_URL=http://di:6080
     - DI_MDN_API_URL=http://di:6081
     - AUTH= on
     -IDLE_SESSION_TIMEOUT = 60
    extra_hosts:
     - “di:172.31.47.51”

Environmental Variables

Ensure that the following environmental variables are set before running SpaceDeck.

AUTH - set to on to enable authentication.

API_URL - set to the url of the api (ie. http://localhost:8090/v2).

DI_API_URL - set to the url of the di api (ie. http://localhost:8090/v2).

DI_MDN_API_URL - set to the url of the mdn api (ie. http://localhost:8090/v2).

IDLE_SESSION_TIMEOUT - how long idle time in minutes before logging out, default 30.

Operation

After GigaSpaces is deployed, SpaceDeck is accessible by port 3000.

Example:

kubectl port-forward spacedeck.<namespace> 3000:3000

SpaceDeck is now available now in browser by open the following URL: http://localhost:3000/ (edited)

Use the GigaSpaces URL with port 3000 to start SpaceDeck.
