 Overview

This topic describes the use case of deploying a cluster on demand to run in the cloud, using the Amazon Elastic Container Service for Kubernetes (Amazon EKS). There are three main steps to this process:

  • Creating the Kubernetes cluster in the cloud
  • Automating the Kubernetes infrastructure by installing the Kubernetes dashboard and Helm
  • Automating the KubeGrid deployment for with Helm.

You can deploy the Kubernetes dashboard in Amazon EKS, so you can properly administer your Kubernetes-based GigaSpaces application. See the Kubernetes Web UI for instructions.

Deploying on Demand

Creating a Kubernetes Cluster

This use case utilizes eksctl (a well-known command-line utility that is used to create and manage Kubernetes clusters on Amazon EKS) to deploy the Kubernetes cluster. You can deploy the cluster following the instructions in Getting Started with eksctl, and create a cluster as in this sample command that creates a cluster with 30 c4.xlarge nodes and prevent updating current context in ~/.kube/config.

eksctl create cluster --name=cluster-6 --region=us-east-1 --nodes=30 
--node-type=c4.xlarge --set-kubeconfig-context=false

For more information about the eksctl utility, see the Weaveworks README on GitHub.

Deploying the Data Grid

The Helm package manager is used for installing in the Kubernetes environment. After installing Helm deploy the cluster with the following steps.

Deploying the Helm Service

To deploy the Helm Tiller service with access to manage resources in your cluster, create a service role for Tiller to use and then type the following command:

helm init --service-account tiller

Deploying KubeGrid

To deploy KubeGrid and the InsightEdge demo:

  1. Type the following command to access the GigaSpaces Helm charts and unpack them locally to a folder named insightedge:

    helm fetch gigaspaces/insightedge --untar

  2. After adding the GigaSpaces Helm charts, type the following command to deploy demo on Amazon EKS

    helm install insightedge --name demo

For more information about InsightEdge KubeGrid deployment options, see the Deploying a Data Grid in Kubernetes topic.

Deleting the Cluster

When done working with the cluster, type the following command to delete the application:

helm delete --purge demo

Then type the following command to completely delete the cluster:

eksctl delete cluster --name=cluster-6 --region=us-east-1
