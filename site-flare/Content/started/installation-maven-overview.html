Maven Artifacts

Maven is a tool used to automate and simplify the development cycle of any Java-based project. The plugin for Maven utilizes Maven to simplify the development cycle of -based applications. This section describes the Maven artifacts that are provided, and how to get started with using Maven.. It also describes how to install and use the XAP Maven plugin. Installation and usage of the XAP plugin for maven.

Installing Maven Artifacts

Prerequisites

Maven must be installed and included in your system PATH variable.

To install the artifacts run the gs host run-agent --autoscript from the product's bin folder.

If you don't have Maven installed, refer to the Apache Maven documentation for download instructions.

Installation

To install the Maven artifacts run the gs maven installscript from the product bin folder. The product config/bin folder is created with

Installation Options

To customize Maven installation for your environment you can create a target folder by:

  • Generate the install script and not run the install, by running the script with an option - gs maven install --generate-only.

    The product config/maven/installer folder is created with the pom xml files.

  • Generate the targetfolder by running the mvn install script..

You can customize the artifacts created by editing the product config/maven/gs-artifacts.txt file.

Maven Repository

artifacts are not currently published in the Maven Central Repo, so you must first configure a repository:

<repository>
   <id>org.openspaces</id>
   <url>http://maven-repository.openspaces.org</url>
</repository>

Artifacts

The main dependency required to use is xap-openspaces.

<dependency>
  <groupId>org.gigaspaces</groupId>
  <artifactId>xap-openspaces</artifactId>
  <version>[%=Versions.maven-version%]</version>
</dependency>
