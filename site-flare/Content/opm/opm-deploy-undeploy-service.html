Deploy and Undeploy Commands in Ops Manager

Ops Manager can be used to deploy and undeploy a service.

Deploy Command in Ops Manager

On the Services Overview screen, click on the plus sign:

Then we see the Deploy Service screen:

  • Service name - Any name
  • Resource - Location of the Java jar file for the service. Can be in a local file, or point to a URL.
  • Properties - Freestyle, name and value. Click on + to add a property and value, delete (or minus sign?) to remove.
    If for example you add a property named "color" and a property value of "red", then this name-value pair will be available to the Java programmer in a configuration file.
  • Topology - Click on the Topology dropdown to choose the service type:

If you click on stateful, you see this screen:

Note for RAM units -- It's not clear right now if Kubernetes will be supported in GA. If it is supported, then the RAM units for Kubernetes environment should automatically show as MiB (Mebibytes), not MB.

If in the previous screen you click on stateless, you see this screen:

In this screen, choose the resources that you wish to make available for the service. Note that there is no High availability for a stateless service.

 

After you click Apply, a message appears in the screen banner, indicating that the service deployment is in progress:

 

When the service deployment is completed, a message indicating this fact appears in the screen banner.

Undeploy Command in Ops Manager

On the Services Overview screen, click on the service card for the service that you wish to undeploy. The Service Detail screen will appear.

Click on Undeploy to undeploy the service.

After you click on Undeploy, it will ask for confirmation, then will undeploy the service.
