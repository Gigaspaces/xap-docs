Ops Manager

The Ops Manager is an enterprise-grade management platform that can be implemented for -based applications across on-premise, cloud, and Kubernetes environments. Ops Manager is a robust web interface that provides visibility and real-time information about your clusters and their services.

Using Ops Manager, you can:

  • View the status of services and microservices to identify healthy and unhealthy clusters
  • Monitor system performance in terms of CPU, latency, IOPS and memory usage metrics
  • Drill down to service instances to pinpoint performance issues
  • Analyze alert trends and volume filtered by time, type, etc.
  • Download service logs for root-cause analysis

Services

A service is also known as a Processing Unit in products. The following types of services are supported.

Icon Name Description
  Stateless contains one or more microservices. This kind of service may act like a client that interacts with other services.
  Stateful contains data in a Space instance. A set of stateful services comprise a data grid.
  Mirror a mirror service is stateless and provides asynchronous persistence to ensure that data isn't lost. The mirror acts as a dispatcher, pushing all primary Space partition changes to the database (or any other data source).
  Web allows deploying web applications using packaged or exploded WAR files.
