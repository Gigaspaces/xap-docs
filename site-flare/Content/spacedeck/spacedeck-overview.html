GigaSpaces SpaceDeck

What is SpaceDeck?

SpaceDeck is an intuitive, streamlined user interface to manage and control the GigaSpaces environment.

Using SpaceDeck, you can define the tools to bring legacy System of Record (SoR) databases into the in-memory data grid that is the core of the GigaSpaces system.

SpaceDeck also includes a set of command line interface (CLI) tools to manage the GigaSpaces system.

SpaceDeck has the following features:

  • No-code functionality – the environment can be set up and monitored using the user-friendly SpaceDeck screens, without the necessity of writing or customizing code

  • Crystal-clear, step-by-step instructions for defining and running the online environment

  • A full complement of data and system monitoring tools, built into the environment

This introductory version of SpaceDeck does not cover all functionality and is intended as an introduction to the new UI and a supplement to our existing Ops Manager, GigaSpaces Management Center and Web Management Console UIs. SpaceDeck is initially available for SmartDIH installations, and will be extended in future releases to eventually replace the older UIs.

Who Can Use SpaceDeck?

SpaceDeck can be used by various user types ("personas") as follows:

System Integrator – define the data in the System of Record (SoR) that will be available to application systems

Data Engineer and Business Analyst – perform queries on application data, spot bottlenecks, and optimize data flow

System Admin – manage the day-to-day operation of the system, and perform changes to fine-tune system operations

Why Do I Need SpaceDeck?

  • If you are running applications that require access to heterogeneous data that is spread across various Systems of Record – you need to gather that data into a unified data model that is not dependent on any particular database or operating system,

  • And if you want to make that data instantly accessible – including querying and updating – in a proven in-memory data grid that is scalable, automatically backed up, designed for the Kubernetes environment, and has easy access to microservices, then you need GigaSpaces Smart DIH.

  • And if you want to design and manage that data using a no-code, elegant, beyond state-of-the-art user interface, then you need SpaceDeck.

What Actions Do I Take with SpaceDeck?

Using SpaceDeck, you can define and modify which data will be pulled from the System of Record and placed in the GigaSpaces in-memory data grid. This data will then be available for fast read/write access. Note that the in-memory data is constantly synchronized with the SoR data.

This definition process includes defining the Data Sources (databases), specifying which tables and fields will be pulled in-memory, and includes additional options to minimize memory footprint and prioritize high-use data.

SpaceDeck is also used to monitor the GigaSpaces system, viewing transaction times and counts, spotting data bottlenecks, and ensuring the smooth 24/7 operation of the system.

Using SpaceDeck, you can define the following items:

  • Data Source – the database in the System of Record that will be used as a source and target of data for the Data Pipeline

  • Data Pipeline – the mechanism to transfer data to and from selected tables in the Data Source to selected Space objects in the GigaSpace in-memory data grid

SpaceDeck also provide monitoring and query screens to view the system status:

  • Spaces Status screen – show status and real-time monitoring of the Spaces in the data grid

  • Data Pipeline Monitoring screen – view Data Pipeline statuses, and stop/start pipelines

  • Data Query screen – perform SQL queries against the data in a Space
