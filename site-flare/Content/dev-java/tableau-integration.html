Integrating with Tableau

This topic describes how to integrate Tableau with using the JDBC connector to retrieve data for visualization and business intelligence needs.

Prerequisites

Your host machine must have the following installed:

  • Java 8 or higher
  • Tableau version 2019.1 or higher

Integrating InsightEdge with Tableau in a Windows environment

The Tableau integration creates a JDBC connector for Tableau, so that Tableau Desktop can query the data grid directly using the JDBC driver. The integration is available in the <product>/tools/tableau folder of the software package. It contains the following files:

  • gs-tableau.bat - runs the setup process for the - connector.
  • gs-tableau-env - environment variables that can be used to customize certain default values for the setup process.
  • gs-tableau-package - script that creates a standalone connector package.

Setting up the - Connector for Tableau

The -connector is based on the Tableau Connector SDK and JDBC integration. The setup process includes providing proprietary requirements about various file locations and execution arguments. provides a script that automates the setup process and starts Tableau Desktop with the - connector enabled.

To implement the -Tableau integration, run the <product>/tools/tableau/gs-tableau.bat script.

The first time this script runs it requires elevated permissions, because it automatically packages and copies the JDBC driver to Tableau's installation files, which generally reside in the Program Files folder.

Customizing the Setup Process

If necessary, you can customize some of the default setup process values, such as the location of the Tableau installation folder or the JDBC driver heap size. To do this, open the product>/tools/tableau/gs-tableau.env script and edit the required environment variables.

Creating a Standalone Connector Package

If your organization includes Tableau users who aren't developers (for example, data scientists that don't need installed locally) you can provide a standalone connector package so that these users can integrate the - connector with their Tableau installation and perform the necessary analytics on the data.

To create the connector package, run the gs-tableau-package script from the <product>tools\tableau folder. This package can then be distributed and installed on any machine to enable accessing data using Tableau.

Integrating InsightEdge with Tableau in a MAC environment

The Tableau integration creates a JDBC connector for Tableau, so that Tableau Desktop can query the data grid directly using the JDBC driver.

Prerequisites

Your host machine must have the following installed:

  • Maven Artifacts

Setting up the - Connector for Tableau

To implement the -Tableau integration:

  • If the JDBC driver is not yet created, run the command <product>/tools/jdbc/build-jdbc-client.sh to create the insightedge-jdbc-client.jar file.
  • Run the command cp <product>/tools/jdbc/insightedge-jdbc-client.jar <TABLEAU_HOME>/Drivers to copy the driver to the Tableau drivers folder.
  • Run the Tableau application with pointer to the tableau folder as in this sample command "/Applications/Tableau Desktop 2019.2.app/Contents/MacOS/Tableau" -DConnectPluginsPath=$IE_HOME/tools/tableau

Connecting to the Space

To configure the data source after Tableau is started:

  1. Click Connect in the Tableau desktop, and select Gigaspaces InsightEdge as the data source.

    If Gigaspaces InsightEdge does not appear in the Connect menu on first access ,click More and select Gigaspaces InsightEdge from the alphabetic list.

  2. In the Gigaspaces InsightEdge window, configure the options as follows:
    1. Type the Server host name or IP address. Include the port number if the Space is not using the default port.
    2. (Optional) If your system includes AnalyticsXtreme, you can type True for Enable AnalyticsXtreme.
    3. Type the Space name.
    4. (Optional) Add Advanced Parameters for the JDBC connection as key-value pairs.

    5. If your system is configured with a secure Space, type the Username and Password.

      For more information about secure Spaces, see the Security section of the Administration guide.

  3. Click Sign In to connect to the data grid.

  4. Verify that Tableau can see the tables in your Space.

    If your data source uses POJOs, they must be configured for java.sql.Date rather than java.util.Date.

Querying the Space using Tableau

After the setup process is complete and the connection has been configured, you can execute queries against the data grid.
